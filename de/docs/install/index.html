EN RU DE

Installation

Obwohl MidnightBSD eigentlich für Anfänger und fortgeschrittene Benutzer attraktiv sein soll, ist die aktuelle Installation für *NIX-Neulinge zu schwierig. Erfahrungen mit BSD, Linux oder einem anderen *NIX artigen System sind sehr hilfreich. FreeBSD Benutzer, die mit sysinstall vertraut sind, sollten keine Probleme bei der Installation haben. Des weiteren bietet das FreeBSD Handbuch zusätzliche Informationen, die nicht auf dieser Webseite stehen.

Vorbereitung der Installation

Sie müssen MidnightBSD von unserem FTP Server oder einem der Mirror herunterladen. Für die Installation benötigen Sie Disk 1 eines Releases oder Snapshots.

Sie werden etwas freien Platz auf Ihrer Festplatte benötigen. Obwohl das System mit wenigen Gigabytes Plattenplatz sehr gut läuft empfehlen wir für die Verwendung von MPorts auf einem Desktop-System mindestens 15GB. Es wird ein PC der Intel Pentium-Klasse empfohlen. Die Unterstützung für 486 wurde aus der Standardinstallation entfernt, sie kann jedoch mit einkompiliert werden. Für die Installation werden mindestens 48MB RAM benötigt.

Verschiedene Grafikkarten werden unterschiedlich gut unterstützt. Des weiteren werden viele Netzwerkkarten und einn paar WLAN-Karten unterstützt. Eine Liste wird später noch angegeben. Es werden neben vielen Onboard-Soundkarten auch die Creative Labs Sound Blaster Audigy und Audigy 2 Familien unterstützt.

MidnightBSD kann neben Microsoft Windows oder Linux auf derselben Festplatte installiert werden. Beim Einsatz von MidnightBSD und Linux wird die Verwendung von GRUB empfohlen. Beim Einsatz von MidnightBSD und Windows wird der in Windows enthaltene Bootmanager empfohlen (Windows XP wurde getestet). Bootmanager von Drittherstellern wie z.B. System Commander 8.20 werden ebenfalls unterstützt.

Sie können MidnightBSD ebenfalls in einer Virtualisierungssoftware wie qemu, Microsoft Virtual PC oder VMWare benutzen. Parallels wurde bisher noch nicht getestet.

MidnightBSD installieren

Legen Sie die MidnightBSD CD in Ihr CD-ROM Laufwerk ein und starten Sie Ihren PC. Eventuell müssen Sie in Ihrem BIOS das Booten von CD aktivieren. Auf manchen Systemen gibt es eine Taste, über die man direkt das Medium zum Booten auswählen kann. So kann man beispielsweise auf einigen Systemen von Dell nach dem Einschalten die Taste F12 betätigen und dann aus einem Menü das CD-ROM Laufwerk auswählen.

Beim Booten von der MidnightBSD-CD sollten Sie zuerst eine Menge weißen Textes und danach einen roten und blauen Bildschirm sehen. Falls mehrere Minuten vergehen ohne das etwas passiert, und Sie auch nicht obigen Bildschirm sehen, dann melden Sie dieses Problem bitte. Es könnte ein Bug oder eine Inkompatibilität mit Ihrem System vorliegen, welche wir beseitigen wollen.

Ab diesem Punkt sollten Sie den Anweisungen von sysinstall folgen um die Installation abzuschließen. Den Snapshots und eventuell dem ersten Release wird leider kein X11 (XORG) als fertiges Paket beiliegen. Versuchen Sie bitte nicht dieses zu installieren. Sie würden nur jede Menge Fehlermeldungen bekommen. Wir hoffen dieses Problem mit dem Release von MidnightBSD 0.2 zu beseitigen.

Beispiel einer Hardwarekonfiguration

Hier ist eine Liste von Systemen auf denen MidnightBSD getestet wurde.

Dell Precision 650 (Dual 2.0Ghz Xeon 1GB ECC RAM SCSI und IDE. Onboard Sound, NIC funktioniert.)

Custom Built Intel Pentium D 805 dual core 2.66Ghz w/ Intel DP965LT motherboard. Onboard sound untested, network card works, IDE channels do not work (CD-ROM, etc). Sata controller works in recent snapshots. Sound Blaster Audigy Gamer and Nvidia GeForce PCIe 7300 work as expected.

Custom Built Nvidia nForce 2 w/ sata raid (MSI motherboard) 512mb PC2700 DDR, 2 sata disks in gmirror raid 1 configuration, 1 ide disk for boot partition, onboard sound, Geforce 5200 AGP 8x. Onboard NIC (nvidia maclan) did not work correctly, but an OEM Intel em worked as expected. Some disk corruption occured on the sata raid setup, but determined to be a faulty drive.

Compaq Cyrix based 366Mhz (?) system. X11 was not tested, but all devices including an included 3com 905 NIC worked as expected.

Dell PowerEdge 5300? Xeon 550Mhz PIII 512MB RAM, all scsi system. Works as expected including integrated ATI graphics.

Virtual PC for Windows works as expected. VPC 7 for Mac OS X works except the CDROM drive is "lost" during install. If you can get it installed using ftp install, it will work fine.

VMware player works under Windows Vista.

Early releases were tested on a Sun Ultra 10 3D creator. The system died however.

One of our developers uses qemu often with the system. It is included in mports. I believe he's also run MidnightBSD virtualized inside of OpenBSD.

In general, if hardware worked in FreeBSD 6.0 Release it will most likely work in MidnightBSD. Some drivers were imported from FreeBSD 6.1 beta, 6.1 Release and 6.2 beta.
