Developer Blog

Einführung

MidnightBSD ist ein Desktop-Betriebsystem für x86 kompatible (incl. Pentium® und Athlon™), und bald auch amd64 kompatible (incl. Opteron™, Athlon™64, und EM64T) Architekturen. Es basierte ursprünglich auf FreeBSD 6.1 Beta. Das Ziel dies Projektes ist es, ein leichtes und einfach zu bedienendes BSD bereitzustellen.

MidnightBSD 0.1-RELEASE

6. August 2007 MidnightBSD 0.1-RELEASE ist jetzt verfügbar. Es beinhaltet mehrere Softwarepakete wie ksh, sudo, OpenNTPD, gcc 3.4.4 und BIND 9.3.4 (mit Patch). Auf CD 1 befinden sich neben dem gesamten Basissystem auch X11 (xorg 6.9) Pakete. Auf CD 2 befinden sich einige fertig compilierte Pakete wie GNUstep, bash3, zsh und cups-base. Weitere Pakete gibt es auf unseren FTP-Servern. Weitere Pakete werden in den nächsten Tagen folgen. Für eine Installation wird nur CD 1 benötigt. Dieses Release ist für Enthusiasten, Entwickler und Interessierte gedacht, die mit Fehlern umgehen können. Das Grundsystem ist stabil, jedoch werden an vielen Teilen wie z.B. den Ports Veränderungen vorgenommen.

0.1-RELEASE Errata

Probleme bei der Installation

Dem projectcenter.app Paket fehlen die Abhägigkeiten auf CD 2.
Dem mutt Paket fehlt das abhängige Paket urlview auf CD 2.

Das GNUstep Paket beinhaltet gdnc, welchem die Datei libgcc_s.so.1 fehlt.

Den Paketen bash 3 und gmake fehlt die Datei libintl.so.6. Dieses Problem
kann durch deinstallieren und aus den Ports bauen von gettext behoben
werden. Ein korrigiertes Paket steht auf den FTP-Servern bereit.

Probleme mit dem System

Die Beispieldatei standard-supfile holt CURRENT anstatt RELENG_0_1

Das neue MPorts-System liegt dem RELEASE nicht bei. Viele Ports funktionieren
mit den MPorts, jedoch nicht mit dem alten Port-System. Es wird empfohlen
die Systemquellen mit RELENG_0_1 und die neuesten Stand der MPorts
mittels CVS oder CVSup herunterzuladen.

Verwendung von Virtual PC für Windows:
Falls Sie MidnightBSD in VPC installieren müssen Sie in /boot/device.hints
hint.acpi.0.disabled="1" setzten. Dies wird das Problem mit
"calcru: negative runtime of" beheben.

Sicherheitsmeldungen

2. Mai 2007
CURRENT und STABLE haben beide den Patch für IPV6 Typ 0 Routing Header erhalten. Das Problem ist das IPV6 Routing Header mehrfach über dieselbe Verbindung laufen können.

10. März 2007
Während viele der Änderungen bzgl. DST im letzen Jahr aufgenommen wurden haben wir uns dazu entschlossen, die gesamte tzdata2007c zu importieren. Benutzer, die von Änderungen bei DST betroffen sind, sollten ihre Quellen aktualisieren und erneut bauen. Der Java-Port hat bisher noch keine Aktualisierung bzgl. DST erfahren. Dies wird später nachgeholt.

23. Januar 2007
Ein "symlink"-Exploit wurde im MidnightBSD Jail-System gefunden. Ein Fix dafür steht zur Verfügung. Bitte besorgen Sie sich die neue /etc/rc.d/jail-Datei aus dem CVS. Patches werden bis zum ersten Release nicht erstellt.
