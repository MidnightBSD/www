MNBSD-2014-2: NFS server deadlock triggered by a trusted client

Severity: Unknown

Affected Package: kernel

Summary: NFS server deadlock triggered by a trusted client

Description

An attacker on a trusted NFS client could cause the NFS server to become deadlocked via a lock-ordering issue during file-handle-to-vnode resolution. This results in a denial of service of the NFS server.

Affected Versions

kernel
