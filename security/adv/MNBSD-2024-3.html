MNBSD-2024-3: Unbounded allocation in ctl(4) CAM Target Layer

Severity: Unknown

Affected Package: ctl

Summary: Unbounded allocation in ctl(4) CAM Target Layer

Description

The command ctl_persistent_reserve_out allows the caller to specify an arbitrary size which will be passed to the kernel's memory allocator. A malicious guest could cause a Denial of Service (DoS) on the host.

Affected Versions

ctl

    Specific versions:

    • 3.0.0
    • 3.0.1
    • 3.0.2
    • 3.1.0
    • 3.1.1
    • 3.1.2
    • 3.1.3
    • 3.1.4
    • 3.1.5
    • 3.2.0

    Recommendations

    No workaround is available. Systems not using virtio_scsi(4) or ctld(8) are not affected. Update to MidnightBSD 3.2.1 release by using the normal update procedure.
